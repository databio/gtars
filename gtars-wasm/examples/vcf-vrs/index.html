VCF → VRS allele identifiers (browser harness)

Scaffold demonstrating the full OPFS + Web Worker plumbing for an in-browser VCF→VRS tool. The reference genome is downloaded once into OPFS, a Worker reads bytes from it via a synchronous access handle, and (eventually) hands them to WASM to compute a ga4gh:VA.<digest> per variant.

STUB NOTICE. The OPFS download-once layer, the Worker, drag-drop, progress, and TSV export are real and working. The actual WASM batch conversion (resident RefgetStore + vcf_to_vrs_ids) is not yet built — see gtars_browser_vcf_vrs_plan_v1.md. Until then a labeled stub returns a placeholder/derived id per variant so the wiring runs end-to-end.
